# Tele-Agent - Submission Checklist

Follow these steps sequentially to ensure a smooth App Store submission process.

## Phase 1: Pre-Submission Verification
- [ ] Review App Store Review Guidelines.
- [ ] Ensure privacy policy is hosted at https://teleagent.app/privacy.
- [ ] Verify that all encryption declarations are ready (ITSDR).
- [ ] Test app on physical iPhone and Apple Watch devices.

## Phase 2: Build & Testing
- [ ] Set Version to 1.0.0 and Build to 1.
- [ ] Run full regression test on latest watchOS and iOS.
- [ ] Check for any hardcoded API keys or sensitive data.
- [ ] Create production archive in Xcode.

## Phase 3: Assets & Metadata
- [ ] Upload App Icon (1024x1024 px).
- [ ] Upload 6 iPhone Screenshots (1242x2208 px).
- [ ] Upload 6 Apple Watch Screenshots (if specific UI is target).
- [ ] Copy name, subtitle, and description from `Tele-Agent_Complete_Submission.md`.
- [ ] Add keywords correctly (comma-separated, no spaces).

## Phase 4: Legal & Compliance
- [ ] Complete "App Data Usage" questionnaire in App Store Connect.
- [ ] Provide Privacy Policy URL.
- [ ] Set "Age Rating" based on content.
- [ ] Confirm "Export Compliance" for encryption.

## Phase 5: App Store Connect Setup
- [ ] Create New App record using Bundle ID `com.teleagent.bridge`.
- [ ] Select "Productivity" as primary category.
- [ ] Set Price to "Free".
- [ ] Set Availability to all desired regions.

## Phase 6: Build Submission
- [ ] Distribute Archive from Xcode to App Store Connect.
- [ ] Wait for build processing.
- [ ] Select the processed build in App Store Connect.

## Phase 7: Final Review submission
- [ ] Double-check all metadata for typos.
- [ ] Submit for "App Review".

## Phase 8: Post-Launch
- [ ] Update landing page with "Download on App Store" link.
- [ ] Monitor analytics and crash reports.
